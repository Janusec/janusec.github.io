OAuth2统一身份认证

简介

Janusec Application Gateway支持OAuth2统一身份认证，可使用企业微信、钉钉、飞书扫码登录，包括:

  • 管理后台
  • 配置中启用OAuth2身份认证的应用(用于员工访问内网应用)

配置

如需启用，请在配置文件config.json中开启：

"oauth": {
            "enabled": true,
            "provider": "wxwork",
            ...
}

上述provider中，wxwork表示企业微信，dingtalk表示钉钉，feishu表示飞书。

应用如何获取用户身份

Janusec认证通过后，会在HTTP请求的头部添加两行：

Authorization: Bearer Access-Token
X-Auth-User: UserID

应用可以通过X-Auth-User获取用户身份（企业微信/钉钉/飞书），或者借助Access-Token（企业微信/飞书）获取进一步的信息。

多节点注意事项

OAuth2需要使用回调域名(在config.json中callback配置)，这里请选定并配置一个域名，可以是任意已配置应用中的域名，假设为www.your_domain.com 。

但需要注意的是，DNS解析应确保其对所有应用的解析，均指向同一个网关。如果员工访问的应用，跟回调域名指向不同的网关地址，则OAuth2将不可用。
